================================================================================
PRODUCT REQUIREMENTS DOCUMENT (PRD)
================================================================================

PRODUCT TITLE
Smart Student Management & Performance Analysis System using MongoDB

VERSION: 1.0
DATE: April 22, 2026
STATUS: Draft


================================================================================
OBJECTIVE
================================================================================

To develop a web-based system that manages student records while providing 
performance insights, risk detection, and analytics using MongoDB as the 
backend database. The system aims to go beyond basic CRUD operations by 
enabling data-driven decision-making for student performance tracking.


================================================================================
TARGET USERS
================================================================================

* Teachers - For tracking and managing student performance
* Educational institutions - For institutional-level analytics
* Admin staff - For data management and reporting


================================================================================
TECH STACK
================================================================================

Frontend:  HTML, CSS, JavaScript
Backend:   Python (Flask)
Database:  MongoDB Atlas (via PyMongo)


================================================================================
CORE FEATURES
================================================================================

1. STUDENT MANAGEMENT (CRUD)
   --------------------------
   * Add new student (name, roll number, marks, attendance)
   * View all students in a structured table
   * Edit student details
   * Delete student records


2. SMART GRADE CALCULATION
   ------------------------
   Automatically assign grades based on marks:
   * 90+ → A
   * 75–89 → B
   * 50–74 → C
   * Below 50 → Fail


3. AT-RISK STUDENT DETECTION ⭐
   ----------------------------
   System identifies students needing attention:
   * Marks < 50 OR Attendance < 75% → "At Risk"
   * Otherwise → "Safe"
   
   Displayed visually to highlight critical cases.


4. SEARCH & FILTER SYSTEM
   -----------------------
   * Search by name or roll number
   * Filter by:
     - Grade
     - Attendance range
     - At-risk students


5. PERFORMANCE DASHBOARD 📊
   -------------------------
   Displays:
   * Total number of students
   * Average marks
   * Class topper
   * Pass vs Fail count
   * Average attendance


6. ANALYTICS MODULE
   -----------------
   Separate analytics view:
   * Highest marks
   * Lowest marks
   * Performance distribution


7. DATA EXPORT FEATURE 📤
   -----------------------
   * Export student records as CSV file
   * Enables reporting and external analysis


8. AUTHENTICATION (BASIC LOGIN SYSTEM)
   ------------------------------------
   * Simple login page
   * Restricts access to authorized users


================================================================================
SYSTEM ARCHITECTURE
================================================================================

Flow:
  Frontend (HTML/CSS/JS)
          ↓
  Flask Backend (API & Logic)
          ↓
  MongoDB Atlas (Cloud Database)


================================================================================
DATABASE DESIGN (MONGODB DOCUMENT)
================================================================================

Collection Name: students

Document Structure:
{
  "name": "Vansh",
  "roll_no": 101,
  "marks": 85,
  "attendance": 90,
  "grade": "B",
  "status": "Safe"
}

Field Descriptions:
* name (String) - Student's full name
* roll_no (Integer) - Unique roll number identifier
* marks (Integer) - Total marks obtained (0-100)
* attendance (Integer) - Attendance percentage (0-100)
* grade (String) - Calculated grade (A/B/C/Fail)
* status (String) - Risk status (Safe/At Risk)


================================================================================
FUNCTIONAL FLOW

1. User logs into system
2. Adds/edits student data
3. Backend processes and stores data in MongoDB
4. System calculates grade and risk status

================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

API Endpoints (Flask Routes):
* POST   /api/students        - Add new student
* GET    /api/students        - Get all students
* GET    /api/students/:id    - Get student by ID
* PUT    /api/students/:id    - Update student
* DELETE /api/students/:id    - Delete student
* GET    /api/dashboard       - Get dashboard metrics
* GET    /api/analytics       - Get analytics data
* POST   /api/export          - Export data to CSV

Frontend Pages:
* login.html          - Authentication page
* dashboard.html      - Main dashboard view
* students.html       - Student list and management
* analytics.html      - Analytics and insights
* add_student.html    - Add new student form
* edit_student.html   - Edit student form
